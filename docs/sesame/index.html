SESAME

This library implements the Strong Exponential Substitution Abstract Machine without Erasure (SESAME) described in the paper "IMELL Cut Elimination with Linear Overhead" by B. Accattoli and C. Sacerdoti Coen, submitted at FSCD 2024.

It also implements parsing and pretty printing rules, a checker for term properness, a final garbage collection pass, a REPL for the machine and a few interesting examples.

The machine and garbage collection are implemented in the Sesame.Reduction sub-module. They work on representation of terms and other data structures via graphs, defined in the Sesame.TermGraphs sub-module. Together, these two sub-modules implement what described in the paper. The remaining modules are for the additional functionalities to obtain a usable REPL.

Library sesame

The entry point of this library is the module: Sesame that is made of the following sub-modules:

  • Modules that implement the SESAME machine:

    • Sesame.TermGraphs Data types used to represent terms and states of the SESAME machine.
    • Sesame.Reduction It implements the SESAME machine reduction rules, the main reduction loop and the Garbage Collection functions.

  • Module to check for properness of terms. The SESAME machine assume all terms to be proper.

    • Sesame.Proper It implements the check that a given term is proper, i.e. all multiplicative variables occur linearly and no multiplicative variable occurs free in an exponential box

  • Modules for parsing and pretty-printing terms:

    • Sesame.Parser It implements a parser for terms written in ASCII art.
    • Sesame.Pp Functions to pretty print the terms and other data structures used by the SESAME machine.

  • Modules that provide the REPL and some pre-made tests:

    • Sesame.REPL It implements a REPL loop: the user is asked to enter a term that is parsed, checked to be proper and evaluated, until the user quits the REPL loop.
    • Sesame.Tests It includes a number of examples of terms to be reduced, including examples from a family of terms that show that the number of exponential steps can be exponential in the number of multiplicative steps.
